# Google 検索セントラル ブログ キャッチアップスキル

Google 検索セントラル ブログ（https://developers.google.com/search/blog）の新着記事を取得して、要約付きのMarkdownダイジェストとして保存する。1回の実行につき1つのダイジェストファイルを作成する。

## 実行環境

このスキルはローカルClaude Code（CLI）とデスクトップ版Claudeアプリのコードモードの両方で動作する。利用可能なツールセットが環境ごとに異なるため、Web取得は「優先するツール → フォールバック」を順に試すこと。

- **Web取得**: ブラウザツール（Chrome / Brave 等のMCPサーバ）が利用可能ならそれを最優先（ローカルマシンで動くためサーバ側のネットワーク制限を受けない）。利用不可な場合は `WebFetch` を使う
- **ファイル保存**: 本リポジトリでは `content/catchup/` ディレクトリ配下に保存する（デスクトップ版コードモードではワークスペースフォルダがリポジトリのルートに対応する）

## 最重要: 記事一覧の取得方法

**ブログ一覧ページ `https://developers.google.com/search/blog` から記事一覧を取得してはいけない。** このページは記事一覧をクライアントサイド（JavaScript）でレンダリングする devsite ベースのページのため、`WebFetch` が取得する静的HTMLには記事一覧が十分に含まれないことがある。一覧ページから記事を拾おうとすると時間を浪費する。

**記事一覧は必ずRSS（Atom）フィードから取得する:**

```
https://developers.google.com/search/blog/feed.xml
```

このフィードには各記事の **タイトル・URL・公開日（published/updated）・概要** が構造化された形（Atom）で含まれている。

- フィード内のリンクは言語パラメータが付かない、または `?hl=en` 付きで返ってくることがあるが、各記事は `?hl=ja` を付けると日本語版にアクセスできる。**記事に記録するURL・本文を取得するURLは `?hl=ja` に統一する。**
- 推測で別のフィードURLを作らない。上記の正しいURLを使う。万一 `feed.xml` が 404 を返した場合のみ、ブログページのフッターにある「RSS」リンクから正しいフィードURLを確認する（推測で `/feeds/...` などを作らない）。

## 絶対に守るべきルール

このスキルで最も重要なのは**正確性**。以下は必ず守ること:

1. **一覧ページ（`/search/blog`）を記事取得に使わない。** 必ずRSSフィード（`https://developers.google.com/search/blog/feed.xml`）を使う
2. **取得したページの内容だけを使う。** 自分の記憶や推測で記事・要約・公開日を補完しない
3. **URLはフィード・記事ページに記載のものを使う。** URLを推測しない。記録するURLは `?hl=ja` に統一する
4. **新着記事を漏れなく拾う。** 既存ファイルと重複しない記事はすべて対象にする
5. **新着がなければファイルを作らない。** すべて既存の場合は「新しい記事はありませんでした」と報告して終了する
6. **ブラウザツールで開いたタブは必ず閉じる**（ブラウザツールを利用した場合のみ）

## 実行手順

### 1. RSSフィードの取得

`https://developers.google.com/search/blog/feed.xml` を取得し、掲載されている記事の **タイトル・URL・公開日・概要** を一覧として抽出する。

### 2. 重複チェック（新着記事の特定）

`content/catchup/` ディレクトリ内の既存の `google-search-blog-*.md` ファイルを確認し、すでに記録済みの記事URLを把握する。

```bash
ls content/catchup/google-search-blog-*.md 2>/dev/null
```

フィードの記事のうち、既存ファイルにまだ含まれていない記事（URLが未掲載のもの）を「新着記事」とする。

- 新着記事が1件もない場合は、ファイルを作成せず「新しい記事はありませんでした」と報告して終了する
- 既存ファイルが1つもない場合（初回）は、フィード上の記事をすべて新着として扱う

### 3. 各記事の要約作成

新着記事それぞれについて、`?hl=ja` 付きの記事URLを取得し、本文に基づいて日本語で2〜3行の要約を作成する。

- フィードの概要だけでは要約として不十分な場合があるため、記事本文を取得して要約する
- 記事本文が取得できなかった場合は、フィードの概要をもとに要約する（推測で内容を作らない）
- 読んだ人が「自分のサイト・SEOに影響があるか」を判断できるように、背景や影響度（コアアップデート・仕様変更・ガイドライン更新など）を簡潔に添える

### 4. Markdownダイジェストの作成

新着記事を以下のフォーマットで **1つのファイル** にまとめる。

**ファイル名**: `google-search-blog-YYYY-MM-DD.md`（YYYY-MM-DD は実行日）
**保存先**: リポジトリの `content/catchup/` ディレクトリ

**テンプレート**:

```markdown
# Google 検索セントラル ブログ キャッチアップ: YYYY-MM-DD

> 取得日: YYYY-MM-DD
> ソース: [Google Search Central Blog](https://developers.google.com/search/blog?hl=ja)

## 今回の注目ポイント

[新着記事のうち特に重要なトピック（コアアップデート・ランキング変更・ガイドライン更新など）を3〜5行でまとめる]

---

## 記事一覧

### [記事タイトル]
- **URL**: [記事の hl=ja URL]
- **公開日**: YYYY-MM-DD
- **要約**: [2〜3行で日本語要約]

（新着記事ごとに上記ブロックを繰り返す。公開日の新しい順に並べる）
```

新着記事をすべて漏れなく含める。記事タイトルは元記事のものをそのまま使う（英語タイトルのままでよい）。

このスキル単体ではpushを行わない。push まで自動化したい場合は `frontend-catchup-and-push` スキルを使う（jser.info / This Week in React / Chrome for Developers と合わせて1回でcommit & pushされる）。

### 5. 定期実行について

このスキルは `frontend-catchup-and-push` スキルのステップに組み込まれており、Claude の routine（スケジュールトリガー）による定期キャッチアップの一部として自動実行される。Google 検索セントラル ブログ単独で定期実行したい場合はスケジュールトリガー機能（`create_trigger` 等）で週次の routine として登録できる。

Google 検索セントラル ブログは不定期更新（概ね週数本〜月数本）なので、週1回のスケジュールが適切。

## よくある失敗と対処

| 失敗 | 対処 |
|------|------|
| `/search/blog` 一覧ページを取得して記事が取れない | 一覧ページはJSレンダリング。RSSフィード `search/blog/feed.xml` を使う |
| フィードURLを推測して404 | `https://developers.google.com/search/blog/feed.xml` を使う。404 のときはブログページのフッターの「RSS」リンクで確認 |
| 記事URLが `?hl=en` のまま | `?hl=ja` に差し替えて統一する |
| 既存記事まで重複してダイジェストに入れる | ステップ2の重複チェックを必ず行う |
| 新着ゼロなのに空ファイルを作る | ファイルを作らず「新しい記事はありませんでした」と報告 |
