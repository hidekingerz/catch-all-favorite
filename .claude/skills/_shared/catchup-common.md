# キャッチアップスキル 共通ルール

`.claude/skills/` 配下の各キャッチアップスキル（jser-catchup / twir-catchup / chrome-blog-catchup / google-search-blog-catchup / apple-news-catchup / ios-release-notes-catchup / android-security-bulletin-catchup / android-release-notes-catchup / apple-security-releases-catchup / google-play-news-catchup / claude-code-catchup）に共通する実行環境・ルール・方針をまとめたもの。各スキルの SKILL.md はソース固有の事項だけを記載し、共通部分は本ファイルを参照する。

**各キャッチアップスキルを実行するときは、まず本ファイルを読み、そのうえで SKILL.md のソース固有の手順に従うこと。** 本ファイルと SKILL.md の記述が矛盾する場合は SKILL.md（ソース固有）を優先する。

## 実行環境

各スキルはローカルClaude Code（CLI）・デスクトップ版Claudeアプリのコードモード・リモート実行環境（Claude Code on the web / routine）のいずれでも動作する。利用可能なツールセットが環境ごとに異なるため、Web取得は「優先するツール → フォールバック」を順に試すこと。

- **Web取得**: ブラウザツール（Chrome / Brave 等のMCPサーバ）が利用可能ならそれを最優先（ローカルマシンで動くためサーバ側のネットワーク制限を受けない）。利用不可な場合は `WebFetch` を使う。SKILL.md に追加のフォールバック（Bash + `curl` 等）が定義されている場合はそれに従う
- **ファイル保存**: リポジトリの `content/catchup/` ディレクトリ配下に保存する（デスクトップ版コードモードではワークスペースフォルダがリポジトリのルートに対応する）

## 絶対に守るべき共通ルール

最も重要なのは**正確性**。以下は全キャッチアップスキルで必ず守ること:

1. **取得したページ/フィードの内容だけを使う。** 自分の記憶や推測で項目・URL・日付・バージョン・要約を補完しない
2. **URLは元ページ/フィードに記載されているものをそのまま転記する。** URLを推測して書かない
3. **新着を漏れなく拾う。** 既存ファイルと重複しない項目はすべて対象にする
4. **存在しない記事・項目を作らない。** 一覧に載っていないものをでっち上げず、取得できたものだけを使う
5. **新着がなければファイルを作らない。** すべて既存の場合は「新しい◯◯はありませんでした」と報告して終了する
6. **ページにアクセスできなければそのソースをスキップする。** エラーで全体を止めず、推測で内容を生成せず、スキップした旨を報告する
7. **ブラウザツールで開いたタブは必ず閉じる。** 作業完了後、自分が開いたタブをすべて閉じる。元から開いていたタブには触れない（ブラウザツールを利用した場合のみ）

## 重複チェックの共通方針

`content/catchup/` 内の既存の同プレフィックスのファイルを確認し（例: `ls content/catchup/<プレフィックス>-*.md`）、既存ファイルに未掲載の項目だけを「新着」とする。

- 重複判定のキー（記事URL / バージョン / 日付＋タイトル 等）と初回実行時の対象範囲は、各 SKILL.md の指定に従う
- 新着が1件もない場合は、ファイルを作成せず「新しい◯◯はありませんでした」と報告して終了する

## ファイルの保存とpush

完成したMarkdownファイルは `content/catchup/` に保存する（ファイル名は各 SKILL.md の指定に従う）。**各スキル単体ではpushを行わない。** push・`index.md` 更新・PR作成・auto-merge まで自動化する場合は `frontend-catchup-and-push` スキルを使う。

## 定期実行

各スキルは `frontend-catchup-and-push` スキルのステップに組み込まれており、Claude の routine（スケジュールトリガー）による定期キャッチアップの一部として自動実行される。単独で定期実行したい場合はスケジュールトリガー機能（`create_trigger` 等）で routine として登録できる。適切な実行頻度は各 SKILL.md の「定期実行について」を参照。

## よくある失敗と対処（共通）

| 失敗 | 対処 |
|------|------|
| 記憶で内容・バージョン・URLを補完してしまう | 取得した内容だけを使う。取得失敗時はスキップ |
| 既存項目まで重複してダイジェストに入れる | 重複チェックを必ず行う |
| 新着ゼロなのに空ファイルを作る | ファイルを作らず「新しい◯◯はありませんでした」と報告 |
| 1ソースの取得失敗で全体が止まる | そのソースだけスキップして報告し、他は続行 |
