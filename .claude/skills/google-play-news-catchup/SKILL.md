# Google Play 最新情報 キャッチアップスキル

Google Play の最新情報テーブル（https://support.google.com/googleplay/android-developer/table/12921780?hl=ja）の新着お知らせを取得して、要約付きのMarkdownダイジェストとして保存する。1回の実行につき1つのダイジェストファイルを作成する。

## 実行環境

このスキルはローカルClaude Code（CLI）とデスクトップ版Claudeアプリのコードモードの両方で動作する。利用可能なツールセットが環境ごとに異なるため、Web取得は「優先するツール → フォールバック」を順に試すこと。

- **Web取得**: ブラウザツール（Chrome / Brave 等のMCPサーバ）が利用可能なら**必ずそれを使う**。利用不可な場合のみ `WebFetch` を試す
- **ファイル保存**: 本リポジトリでは `content/catchup/` ディレクトリ配下に保存する

## 最重要: 取得方法の注意

**このソースにはRSSフィードが存在しない。** 情報源は以下の一覧テーブルページのみ:

```
https://support.google.com/googleplay/android-developer/table/12921780?hl=ja
```

このページには各お知らせの **日付・タイトル・説明（および詳細ページへのリンク）** がテーブル形式で含まれている。

- **`support.google.com` は環境のネットワークポリシー次第でアクセスがブロックされる**ことがある。アクセスできない場合は、**このソースをスキップし「support.google.com にアクセスできなかったため Google Play はスキップ」と報告して終了する**（エラーで全体を止めない。推測で内容を作らない）
- URLは `?hl=ja` 付きの日本語版を使う

## 絶対に守るべきルール

このスキルで最も重要なのは**正確性**。以下は必ず守ること:

1. **取得したページの内容だけを使う。** 自分の記憶や推測でお知らせ・要約・日付を補完しない
2. **URLはページに記載のものを使う。** お知らせ内のリンク（詳細ページ）はページに記載のURLをそのまま転記する。URLを推測しない
3. **新着お知らせを漏れなく拾う。** 既存ファイルと重複しないお知らせはすべて対象にする
4. **新着がなければファイルを作らない。** すべて既存の場合は「新しいお知らせはありませんでした」と報告して終了する
5. **ページにアクセスできなければスキップする。** 取得失敗時に記憶で内容を生成しない
6. **ブラウザツールで開いたタブは必ず閉じる**（ブラウザツールを利用した場合のみ）

## 実行手順

### 1. 最新情報テーブルの取得

`https://support.google.com/googleplay/android-developer/table/12921780?hl=ja` を取得し、掲載されているお知らせの **日付・タイトル・説明・詳細ページへのリンク（あれば）** を一覧として抽出する。

- ブラウザツール利用時はページが完全にレンダリングされてから抽出する
- テーブルは日付の新しい順に並んでいる。直近のもの（概ね過去1〜2か月分）を対象にすれば十分

### 2. 重複チェック（新着お知らせの特定）

`content/catchup/` ディレクトリ内の既存の `google-play-news-*.md` ファイルを確認し、すでに記録済みのお知らせ（日付＋タイトル）を把握する。

```bash
ls content/catchup/google-play-news-*.md 2>/dev/null
```

テーブルのお知らせのうち、既存ファイルにまだ含まれていないもの（**日付＋タイトルの組み合わせ**が未掲載のもの）を「新着」とする。このソースはお知らせごとの固有URLがない場合があるため、重複判定はURLではなく日付＋タイトルで行う。

- 新着が1件もない場合は、ファイルを作成せず「新しいお知らせはありませんでした」と報告して終了する
- 既存ファイルが1つもない場合（初回）は、直近1〜2か月分のお知らせを新着として扱う

### 3. 各お知らせの要約作成

新着お知らせそれぞれについて、テーブルの説明文に基づいて日本語で2〜3行の要約を作成する。

- テーブルの説明だけでは不十分で、詳細ページへのリンクがある場合は、リンク先を取得して要約に反映してよい（取得できなければテーブルの説明のみで要約する。推測で内容を作らない）
- 読んだ人が「自分のアプリ・開発に影響があるか」を判断できるように、影響度（ポリシー変更・対応期限・新機能・API要件など）と対応期限があればそれを簡潔に添える

### 4. Markdownダイジェストの作成

新着お知らせを以下のフォーマットで **1つのファイル** にまとめる。

**ファイル名**: `google-play-news-YYYY-MM-DD.md`（YYYY-MM-DD は実行日）
**保存先**: リポジトリの `content/catchup/` ディレクトリ

**テンプレート**:

```markdown
# Google Play 最新情報 キャッチアップ: YYYY-MM-DD

> 取得日: YYYY-MM-DD
> ソース: [Google Play の最新情報](https://support.google.com/googleplay/android-developer/table/12921780?hl=ja)

## 今回の注目ポイント

[新着お知らせのうち特に重要なトピック（ポリシー変更・対応期限・ターゲットAPIレベル要件など）を3〜5行でまとめる]

---

## お知らせ一覧

### [お知らせのタイトル]
- **日付**: YYYY-MM-DD
- **詳細リンク**: [ページに記載があれば転記。なければこの行を省く]
- **要約**: [2〜3行で日本語要約。対応期限があれば明記]

（新着お知らせごとに上記ブロックを繰り返す。日付の新しい順に並べる）
```

新着お知らせをすべて漏れなく含める。タイトルはページのものをそのまま使う。

このスキル単体ではpushを行わない。push まで自動化したい場合は `frontend-catchup-and-push` スキルを使う。

### 5. 定期実行について

このスキルは `frontend-catchup-and-push` スキルのステップに組み込まれており、`schedule` スキルによる定期キャッチアップの一部として自動実行される。Google Play の最新情報は不定期更新（概ね月数本、ポリシー更新時期は集中）なので、週1回のスケジュールが適切。

## よくある失敗と対処

| 失敗 | 対処 |
|------|------|
| ページにアクセスできない（ブロック） | 環境のネットワークポリシーで `support.google.com` が許可されているか確認。ブラウザツールを使う。取得不可ならこのソースだけスキップして報告 |
| RSSフィードを探して時間を浪費する | このソースにRSSはない。テーブルページを直接取得する |
| 記憶でお知らせ内容を補完してしまう | 取得できた内容のみ使う。取得失敗時はスキップ |
| 既存お知らせまで重複してダイジェストに入れる | ステップ2の重複チェック（日付＋タイトル）を必ず行う |
| 新着ゼロなのに空ファイルを作る | ファイルを作らず「新しいお知らせはありませんでした」と報告 |
