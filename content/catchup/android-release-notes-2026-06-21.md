## 今回の注目ポイント

前回キャッチアップ（2026-06-10、Android 16 まで）以降の新着として、**Android 17 リリースノート**（最終更新 2026-06-18）が公開された。アーキテクチャでは ION メモリアロケータが完全に廃止され、AOSP のビルドソースツリーが読み取り専用化される。メモリ管理（Memory Limiter / mmd / PMGD）やロック画面のレート制限強化、Contacts Picker・位置情報インジケータといったプライバシー透明性の向上、VVC デコード対応、自動車向け Scalable UI / SDV など、基盤からプライバシー・メディア・車載まで幅広い変更が含まれる。

## リリースノート一覧

### Android 17
- **日付**: 2026-06-18（ページ最終更新）
- **関連リンク**: [Android 17 release notes](https://source.android.com/docs/whatsnew/android-17-release?hl=ja)
- **要約**:
  - **アーキテクチャ／ビルド**: ION メモリアロケータが 2025 年 12 月でサポート終了となり、ION を使用するベンダーコードは失敗するため完全に削除が必要。Android 17 以降、AOSP のビルドソースツリーは読み取り専用となり、ビルド中に変更が発生するとビルドが失敗する。
  - **オーディオ**: SCO（Synchronous Connection-Oriented）のルーティング管理を Bluetooth フレームワークから Android オーディオフレームワークへ移管。
  - **自動車（Automotive）**: 多パネルの高度な UI を構築できる Scalable UI（システムバー／ナビゲーションバーのカスタマイズ対応）と、ソフトウェア更新で進化する Software-Defined Vehicle（SDV）プラットフォームをサポート。
  - **接続性**: OTT の音声・映像通話を 5G スライス等のプレミアムネットワークインターフェースへアプリ改修なしで自動ルーティング。Out-of-Band 仕様 v2／v3 でセッション中の技術切替やモーション通知に対応。
  - **表示**: ディスプレイ単位のデスクトップウィンドウ（システム装飾対応）、大画面アプリ互換性向上のためのカメラ互換オーバーレイ設定等。
  - **メディア**: VVC（Versatile Video Coding）デコードのフレームワーク／パーサーサポートを追加（ハード／ソフトデコーダは SoC ベンダー提供が前提）。
  - **パフォーマンス・セキュリティ**: メモリ管理を強化（cgroup v2 で監視する Memory Limiter、ZRAM 維持を担う mmd、孤立したリークの全体波及を防ぐ PMGD）。ハードウェアラップキーを KeyMint ではなく Linux カーネル ioctl で簡素化。ロック画面のレート制限 UX 強化、Weaver HAL のプリウォームで認証情報検証の遅延を最大 200ms 削減。
  - **権限／プライバシー**: フル権限なしで選択した連絡先のみ共有できる Contacts Picker、非システムアプリの前面での位置情報アクセスを知らせるステータスバーの位置情報インジケータ。
