## 今週の注目ポイント

- **React チームが Working Group ベースに再編**。React Foundation ガバナンスを反映し、team ページが刷新された。各 Working Group に Leadership Council の代表が置かれ、コアの意思決定体制が変わる大きな動き。
- **Next.js が月次セキュリティパッチ体制へ**。タイムラインと深刻度を事前告知する運用に移行し、セキュリティ対応の計画が立てやすくなる。
- **Bun を Rust で書き直し**。約64並列の Claude インスタンスで Zig 約53.5万行を移植し v1.4.0 をリリース。メモリバグ減・バイナリ約20%縮小という、AI 駆動リライトの注目事例。
- **npm 12.0 でサプライチェーン強化**。install スクリプトがデフォルト opt-in になり、インストール時のコード実行がデフォルトでブロックされる。既存プロジェクトへの影響を要確認。
- **Suspense の発火条件が公式ドキュメントに整理**。lazy ロード・`use()` での promise 読み取り・ストリーミング SSR など、何が Suspense を活性化するかがライブ例付きで解説された。

## React

### Meet the new React Team - Reorganized by Working Groups
- **URL**: https://react.dev/community/team
- **要約**: team ページが刷新され、React Foundation のガバナンスを反映。各 Working Group ごとに Leadership Council の代表とメンバーバッジが示されるようになった。

### React Docs - What activates a Suspense boundary
- **URL**: https://react.dev/reference/react/Suspense#what-activates-a-suspense-boundary
- **要約**: lazy ロード、`use()` での promise 読み取り、スタイルシート/フォント/画像、ストリーミング SSR、遅延 CPU 処理などを扱う新セクション。サスペンド中コンテンツの表示は最大 300ms に1回。

### Next.js Security Release and Our Next Patch Release
- **URL**: https://nextjs.org/blog/next-security-release-program
- **要約**: 月次のセキュリティパッチを計画。タイムラインと深刻度を事前告知し、利用側が計画を立てやすくする運用へ。

### Better Auth is joining Vercel
- **URL**: https://better-auth.com/blog/better-auth-joins-vercel
- **要約**: フレームワーク非依存の認証ライブラリが Vercel に参加。エージェント型ワークフロー向けの「Agent Auth Protocol」へと軸足を移す。

### Cache Components in Next.js 16
- **URL**: https://certificates.dev/blog/cache-components-in-nextjs
- **要約**: App Router がデフォルト dynamic に転換し、`use cache`・cacheLife・cacheTag で opt-in キャッシュ。Instant Navigations 向けに旧 API を置き換える。

### React Compiler explained
- **URL**: https://neciudan.dev/react-compiler-explained
- **要約**: React Compiler の実際の出力、手書きメモ化との違い、有効化手順を解説。

### useOptimistic Already Handles Rollback in Next.js 16
- **URL**: https://dev.to/shubhradev/my-nextjs-16-optimistic-ui-looked-perfect-then-someone-clicked-it-five-times-fast-b2c
- **要約**: トランジション失敗時に自動でベース状態へ戻すため、catch ブロックでの手動ロールバックは多くの場合不要になる。

### HTMX and Web Components Instead of React
- **URL**: https://kore-nordmann.de/blog/htmx-and-web-components-instead-of-react.html
- **要約**: React をデフォルトのフロントエンド解として使うことと、それがオーバーエンジニアリングになるケースを論じる。

### Async hydration in Preact
- **URL**: https://jovidecroock.com/blog/resumed-hydration-preact/
- **要約**: Preact と React のハイドレーションの違いを深掘り。Preact v10 は Suspense 境界で一時停止し、v11 の Hydration 2.0 で信頼性が向上する。

### SolidJS 2.0: A React Developer's First Look
- **URL**: https://morello.dev/blog/solidjs-2-react-developers-first-look
- **要約**: Solid の signals と利点（依存配列不要・stale closure なし・リアクティブグラフでのネイティブ async）を React 開発者視点で概観。

### Exporting Next.js Server Traces with OpenTelemetry
- **URL**: https://blog.sentry.io/nextjs-export-traces-opentelemetry/
- **要約**: Next.js はすでにサーバー側トレースを出力しており、最小の手間で任意の OTLP エンドポイントへ送信できる。

### React Router 8.2
- **URL**: https://github.com/remix-run/react-router/releases/tag/react-router%408.2.0
- **要約**: 非 Node の Framework モード向けに Web Streams デフォルトサーバーエントリを追加。Node 22+ の opt-in フラグ、nub パッケージマネージャーをサポート。

### Storybook 10.5

### Astryx 0.1.3

- **要約**: StyleX で構築された Meta のデザインシステム。テーブル・キーボードナビゲーション・アクセシビリティを改善。

### 動画: Jack Herrington Can't Think of a Reason to Choose Next.js Anymore

## その他（JS / CSS / Web全般）

### TC39 - Agenda for the 115th meeting - July 20-23
- **URL**: https://github.com/tc39/agendas/blob/main/2026/07.md
- **要約**: `Promise.allKeyed()`、`JSON.parseImmutable()`、`Error.code`、`Map.take()`、条件内での宣言、RegExp linear matching など多数の提案が進む可能性。

### pnpm RFC - Native Monorepo Versioning
- **URL**: https://github.com/pnpm/rfcs/pull/18
- **要約**: pnpm にビルトインのリリース管理が登場。changesets では構造的に表現できないパッケージ単位のプレリリースレーンを持つネイティブな代替。今週マージ。

### Rewriting Bun in Rust
- **URL**: https://bun.com/blog/bun-in-rust
- **要約**: 約64並列の Claude インスタンスで Zig 約53.5万行を移植し v1.4.0 をリリース。メモリバグ減・バイナリ約20%縮小。API 費用は約16.5万ドルという AI 駆動リライトの事例。

### Node's phantom dependency problem and the path to a 5x faster package manager
- **URL**: https://nubjs.com/blog/unblocking-the-global-virtual-store
- **要約**: Nub チームがグローバル仮想ストアの課題を解説。symlink/hardlink の組み合わせで warm install が Bun より約5倍速い。ただし Next.js/Turbopack や RN/Metro には別の課題も。

### Fetch Needs Error Codes
- **URL**: https://www.jasnell.me/posts/fetch-needs-error-codes
- **要約**: fetch API が HTTP/2・HTTP/3 のエラーコードを落としている問題を詳細分析。改善に向けた提案が進行中。

### The Siren Song of ariaNotify()
- **URL**: https://css-tricks.com/the-siren-song-of-arianotify/
- **要約**: スクリーンリーダーへ通知を送る新 API。脆い aria-live ハックを置き換えるが、手軽さゆえ alert() のように濫用されやすい点に注意。

### Do we still need build tools?
- **URL**: https://olliewilliams.xyz/blog/no-build/
- **要約**: 一般的なバンドラーのタスクをネイティブな Web 標準がどこまでカバーできるかを検証。複雑なプロジェクトではバンドルに依然利がある。

### Proxy and Reflect
- **URL**: https://piccalil.li/blog/proxy-and-reflect/?ref=main-rss-feed
- **要約**: Valtio や MobX などプロキシベースの状態ライブラリを支える API を解説。

### ツール: Native SDK
- **URL**: https://native-sdk.dev/
- **要約**: TypeScript と宣言的マークアップを単一ネイティブバイナリにコンパイルする Vercel Labs のツールキット。ブラウザ/WebView/JS エンジン/GC 不要。macOS・Linux・Windows・モバイル（実験的）対応。

### npm 12.0
- **URL**: https://socket.dev/blog/npm-12
- **要約**: install スクリプトがデフォルト opt-in に。2FA バイパストークンを非推奨化。preinstall/install/postinstall のインストール時コード実行をデフォルトでブロックし、サプライチェーンを強化。
